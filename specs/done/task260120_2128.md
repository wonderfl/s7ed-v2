# 작업계획서: 공통 슬라이더 적용 시 원본 중앙 포인트 전달 누락 수정

## 작업 목표
`_apply_common_sliders_to_landmarks` 함수에서 `morph_face_by_polygons` 호출 시 원본 중앙 포인트(`left_iris_center_orig`, `right_iris_center_orig`)를 전달하지 않아 발생하는 문제를 수정합니다.

## 문제 분석

### 발견된 문제
1. **첫 번째 `morph_face_by_polygons` 호출** (`gui/face_edit/morphing/logic.py` 843줄):
   - 원본 중앙 포인트 파라미터를 전달하지 않음
   - 결과: `원본 중앙 포인트: left_orig=None, right_orig=None`
   - 경고: "원본 왼쪽 중앙 포인트 계산 실패, 변형된 값을 원본으로 사용"
   - 랜드마크 변형 확인: 최대 차이=0.00픽셀, 변경된 포인트=0개

2. **두 번째 `morph_face_by_polygons` 호출** (`gui/face_edit/polygon_drag_handler.py` 672줄):
   - 원본 중앙 포인트를 제대로 전달함 (`left_iris_center_orig`, `right_iris_center_orig`)
   - 결과: 정상 동작, 랜드마크 변형 확인: 최대 차이=11.68픽셀, 변경된 포인트=1개

### 문제 원인
- `_apply_common_sliders_to_landmarks` 함수에서 `morph_face_by_polygons` 호출 시 원본 중앙 포인트 파라미터를 전달하지 않음
- `apply_polygon_drag_final` 함수에서는 `landmark_manager`에서 원본 중앙 포인트를 가져와서 전달하지만, `_apply_common_sliders_to_landmarks`에서는 누락됨

### 영향
- 첫 번째 호출에서 원본 중앙 포인트가 None이 되어 계산을 시도하지만 실패
- 변형된 값을 원본으로 사용하여 실제 변형이 없어도 변형이 발생한 것처럼 처리될 수 있음
- 불필요한 계산 및 경고 메시지 발생
