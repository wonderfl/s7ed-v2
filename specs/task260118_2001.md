# 작업계획서: Size 슬라이더를 Size X, Size Y로 분리

## 작업 목표
전체탭의 Size 슬라이더를 Size X와 Size Y로 분리하여 가로와 세로 크기를 독립적으로 조절할 수 있도록 합니다.

## 문제 분석
1. **현재 상태**: 하나의 `region_size` 슬라이더가 x와 y 모두에 동일하게 적용됨
2. **제한사항**: 가로와 세로를 독립적으로 조절할 수 없음
3. **요구사항**: Size X와 Size Y를 별도로 조절하여 더 정밀한 편집 가능

## 작업 계획

### 1. 변수 추가 (__init__.py)
- **위치**: `gui/face_edit/__init__.py`
- **작업**: 
  - `region_size` 변수를 `region_size_x`, `region_size_y`로 분리
  - 기본값은 모두 1.0으로 설정
- **예상 효과**: X와 Y 크기를 독립적으로 제어 가능

### 2. UI 슬라이더 분리 (slider_ui.py)
- **위치**: `gui/face_edit/slider_ui.py`
- **작업**:
  - "Size:" 슬라이더를 "Size X:", "Size Y:" 두 개로 분리
  - 각각 `region_size_x`, `region_size_y` 변수에 연결
- **예상 효과**: 사용자가 X와 Y를 독립적으로 조절 가능

### 3. 로직 수정 (logic.py)
- **위치**: `gui/face_edit/morphing/logic.py`
- **작업**:
  - `size` 변수를 `size_x`, `size_y`로 분리
  - `adjust_region_size` 함수 호출 시 `size_x`, `size_y` 전달
  - 조건 확인도 `size_x`, `size_y` 각각 확인
- **예상 효과**: X와 Y 크기가 독립적으로 적용됨

### 4. 조정 함수 수정 (region_adjustments.py)
- **위치**: `utils/face_morphing/adjustments/region_adjustments.py`
- **작업**:
  - `adjust_region_size` 함수의 `size_ratio` 파라미터를 `size_x`, `size_y`로 분리
  - `new_width`, `new_height` 계산 시 각각 `size_x`, `size_y` 사용
- **예상 효과**: X와 Y 크기가 독립적으로 적용됨

## 작업 완료 내역
- [x] __init__.py에 region_size_x, region_size_y 변수 추가
- [x] slider_ui.py에서 Size 슬라이더를 Size X, Size Y 두 개로 분리
- [x] logic.py에서 size_x, size_y를 adjust_region_size에 전달하도록 수정
- [x] adjustments/region_adjustments.py의 adjust_region_size 함수가 size_x, size_y를 받도록 수정

## 예상 효과
- **정밀도 향상**: 가로와 세로를 독립적으로 조절 가능
- **편집 자유도 증가**: 더 다양한 얼굴 형태 편집 가능
- **사용자 경험 개선**: 더 직관적인 크기 조절

## 주의사항
- **하위 호환성**: 기존 코드에서 `region_size`를 사용하는 부분이 있는지 확인 필요
- **기본값**: size_x와 size_y 모두 1.0으로 설정하여 기존 동작과 동일하게 유지
- **조건 확인**: size_x와 size_y 중 하나라도 기본값이 아니면 적용
